# Same-Model-Different-Morals-How-Ethical-Constitutions-Shape-LLM-Refusal-Boundaries
PRISM research. 
Presenting at 2026 CSSU Undergrad Research Conference on March 24th, 2026. 

Abstract: 
Constitutional AI is an alignment technique in which a model is guided by a constitution, that is, a set of ethical principles that shape how it responds. Anthropic’s original approach embedded these principles using training-time reinforcement learning, but retraining requires significant compute and effort. A more accessible alternative is to insert the constitution into the system prompt at inference time. This approach is shallower than training-time alignment since it cannot reshape the model’s internals, but it is the primary tool available to practitioners deploying open-weight models without resources to retrain. 
This study asks whether the philosophical grounding of such a constitution matters for safety. Language models can fail in two opposite directions: over-refusal, where the model declines a request it should answer, and under-refusal, where it complies with a request it should refuse. Even when a model does comply with a harmful request, the severity of the output varies, which we measure as toxicity. A complete safety evaluation must capture both refusal behavior and output toxicity. We test three constitutions built on different ethical frameworks (deontological, utilitarian, and virtue ethics) plus a no-constitution baseline across two open-weight models, measuring compliance and refusal rates across 500 benchmark prompts (OR-Bench, Do-Not-Answer). Using 300 adversarial prompts (HarmBench), we score the toxicity of every non-refused response with the Perspective API. No training or fine-tuning is performed. The results will clarify whether ethical framework choice produces measurable differences in refusal patterns and output toxicity, informing how practitioners design safety guidelines for deployed models.
